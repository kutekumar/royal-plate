STEP 1: Run Database Migration (5 minutes)
────────────────────────────────────────────────────────────────────────────

1. Open Supabase Dashboard (supabase.com)
2. Navigate to SQL Editor
3. Copy and run: supabase/migrations/add_restaurant_coordinates.sql

   Or run this SQL:

   ALTER TABLE restaurants
   ADD COLUMN IF NOT EXISTS latitude DECIMAL(10, 8),
   ADD COLUMN IF NOT EXISTS longitude DECIMAL(11, 8);

   CREATE INDEX IF NOT EXISTS idx_restaurants_location
   ON restaurants(latitude, longitude);


STEP 2: Add Coordinates (2 minutes - Quick Test)
────────────────────────────────────────────────────────────────────────────

Option A: Quick Test (Recommended)
   • Copy and run: supabase/migrations/add_sample_coordinates.sql
   • This adds coordinates to first 10 restaurants
   • Perfect for testing the map feature

Option B: Manual Entry
   • Go to Table Editor → restaurants
   • Add latitude/longitude for each restaurant
   • Find coordinates at latlong.net

Option C: Automatic Geocoding
   • Update credentials in scripts/geocode-restaurants.ts
   • Run: npx tsx scripts/geocode-restaurants.ts
   • Automatically geocodes all addresses


STEP 3: Test the App (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Start development server:
   npm run dev

2. Open http://localhost:5173/home

3. Look for "Explore Map" section

4. Click "Show" button

5. You should see:
   ✓ Interactive map with restaurant markers
   ✓ Royal Plate logo on markers
   ✓ Click markers to navigate
   ✓ Real distances displayed

6. Click any restaurant card

7. Scroll to address section

8. Click "Show on map"

9. You should see:
   ✓ Individual restaurant location
   ✓ Route line from you to restaurant
   ✓ Distance visualization
